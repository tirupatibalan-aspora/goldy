# Goldy Status Report Template

This is the canonical template for all Goldy status reports — terminal, markdown, and Slack.
The script `scripts/generate-report.sh` follows this structure exactly.

## Template Structure

```
GOLDY STATUS REPORT
<Day>, <Date> at <Time>
------------------------------------------------------------

<project-name> | <Platform> | <branch>
------------------------------------------------------------

YESTERDAY
  - <commit message> (<hash>)
  - <commit message> (<hash>)

TODAY
  - <commit message> (<hash>)
  - <commit message> (<hash>)
  - Stats: <files changed, insertions, deletions>

BLOCKERS
  - <uncommitted files, unpushed commits, merge conflicts>
  - None (if clean)

BUILD
  - Latest version: <tag>
  - Branch: <branch> (<N> commits)
  - Last activity: <date time>

------------------------------------------------------------

(repeat per project)

CROSS-PLATFORM ALERTS
  - <shared contract changes, API drifts>

SUGGESTIONS
  - <stale branches, large uncommitted diffs, old TRUTH.md>

REVIEW BOT
  - <reviewer>: <N> patterns (<critical>, <major>, <minor>)

OPEN PRs
  - <project>: #<number> <title> (<status>)

INFRASTRUCTURE
  - <project>: <N> total changelogs (<N> today)
  - Shared memory: <N> files
  - TRUTH.md: active/missing

------------------------------------------------------------
Generated by Goldy | <date> <time>
```

## Formatting Rules

- **Bold** for all section headers and key labels
- **Bullets** for all list items (indented with 2 spaces)
- **Separators** (`----`) between sections
- **No italics** — never use _italic_ or *single asterisk*
- **No emojis** — plain text only
- **Backticks** for branch names, hashes, and commands
- **Consistent** — same structure every time, regardless of output format

## Slack Adaptation

- `**bold**` converts to `*bold*` (Slack mrkdwn)
- Separator converts to `───────────────────────────────`
- All other formatting preserved

## Sections (in order)

1. **Header** — report title, date, time
2. **Per-project block** (repeated for each detected repo):
   - Yesterday — commits from previous day
   - Today — commits from current day + file stats
   - Blockers — uncommitted, unpushed, conflicts
   - Build — version tag, branch info, last activity
3. **Cross-Platform Alerts** — from TRUTH.md
4. **Suggestions** — auto-detected (stale branches, large diffs)
5. **Review Bot** — per-reviewer pattern counts by severity
6. **Open PRs** — from GitHub (via gh CLI)
7. **Infrastructure** — changelog counts, memory files
8. **Footer** — generated by line

## Pre-Push Rules (STRICT — enforced by Goldy)

Every `git push` MUST follow these 3 steps in order — no exceptions:

1. **Aspora Review Bot audit** — audit diff against all reviewer learnings for the target platform. Min score: 8/10. Fix all critical/major issues.
2. **Run local tests** — run test suites for changed files. CI is for verification, not discovery.
3. **Push** — only after steps 1-2 pass.

These rules apply to ALL modules and ALL team members using Goldy. The report should flag any push that skipped these steps as a blocker/suggestion.
